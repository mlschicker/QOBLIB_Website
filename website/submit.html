Submit Results

Build a valid QOBLIB submission for any number of instances across any number of problems in one go. Everything is checked live in your browser, then exported as ready-to-commit files. A GitHub Action re-runs the authoritative checker (including solution feasibility) when you open the pull request.

  1. 1 Shared details
  2. 2 Add your results
  3. 3 Fix any flagged errors
  4. 4 Export & open a PR
1 · Shared details applied to every row — fill once
2 · Results one row per instance

Fields are required unless tagged (optional). For required numeric fields that do not apply to your run (e.g. QPU runtime for a classical method, or an optimality bound you did not compute), enter N/A rather than leaving them blank.

3 · Validation —

Nothing is flagged until you press Check submission. After that, highlights update live as you fix them.

4 · Export

Download a ZIP that mirrors the repository layout, unzip it at the root of your QOBLIB fork, drop your <instance>_solution.<ext> file next to each _summary.csv, then open a pull request. Files are still produced if errors remain, so you can iterate — but the PR check will fail until they are resolved.
